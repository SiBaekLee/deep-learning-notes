## 1. 핵심 공식

입력의 한 변 크기를 W, 필터 크기를 F, 패딩을 P, 스트라이드를 S라 할 때, 출력 feature map의 한 변 크기 O는:

$$
O = \left\lfloor \frac{W - F + 2P}{S} \right\rfloor + 1
$$

가로(W)와 세로(H)에 각각 동일하게 적용한다.

$$
O_{width} = \left\lfloor \frac{W - F + 2P}{S} \right\rfloor + 1, \qquad
O_{height} = \left\lfloor \frac{H - F + 2P}{S} \right\rfloor + 1
$$

- ⌊ ⌋ 는 내림(floor). 나눗셈이 딱 떨어지지 않으면 버린다.
- 출력의 **채널 수(깊이)는 필터의 개수 K**와 같다(아래 5절 참고).

---

## 2. 각 요소의 의미

| 기호 | 이름 | 의미 |
| --- | --- | --- |
| W (H) | 입력 크기 | 입력 이미지/feature map의 가로(세로) 픽셀 수 |
| F | 필터(커널) 크기 | Convolution 필터 한 변의 크기 (예: 3×3 → F=3) |
| K | 필터 개수 | 사용하는 필터 수 = **출력 채널 수** |
| P | 패딩 | 입력 가장자리에 덧대는 픽셀 수 (한쪽 기준) |
| S | 스트라이드 | 필터가 한 번에 이동하는 칸 수 |

---

## 3. 공식이 나오는 원리

공식의 각 항이 어디서 오는지 단계적으로 이해하면 외우지 않아도 된다.

**(1) `W − F` — 필터가 들어갈 수 있는 이동 거리**

필터는 입력 밖으로 나갈 수 없다. 크기 W인 입력 위에 크기 F인 필터를 올리면, 필터의 시작 위치는 0부터 `W − F`까지 갈 수 있다. 즉 필터가 미끄러질 수 있는 총 거리가 `W − F`이다.

**(2) `+ 2P` — 패딩으로 늘어난 입력**

패딩을 한쪽에 P씩, 양쪽에 더하면 입력의 유효 크기가 `W + 2P`가 된다. 따라서 이동 가능 거리는 `(W + 2P) − F`로 늘어난다.

**(3) `÷ S` — 스트라이드만큼 건너뛰기**

필터가 1칸씩이 아니라 S칸씩 이동하므로, 가능한 이동 횟수는 거리를 S로 나눈 값이다: `(W − F + 2P) / S`.

**(4) `+ 1` — 시작 위치(첫 칸)**

이동 "횟수"에 더해, 맨 처음 놓인 위치 자체도 출력 한 칸을 만든다. 그래서 마지막에 1을 더한다.

**(5) `⌊ ⌋` — 내림**

S로 나눠 떨어지지 않으면 필터가 끝까지 닿지 못하는 마지막 자투리 영역이 생긴다. 이 부분은 출력으로 세지 않으므로 내림 처리한다.

> 정리: **(이동 가능 거리 ÷ 보폭) + (시작 칸)** = ⌊(W − F + 2P) / S⌋ + 1

## 4. 패딩(Padding) — Valid vs Same

| 방식 | 패딩 값 | 출력 크기 | 특징 |
| --- | --- | --- | --- |
| **Valid** | P = 0 | 입력보다 작아짐 | 패딩 없이 유효 영역만 계산 |
| **Same** | P = (F − 1) / 2 (S=1일 때) | **입력과 동일** | 출력 크기를 입력과 같게 유지 |

`Same` 패딩은 스트라이드 S=1일 때 출력 크기가 입력과 같아지도록 패딩을 맞추는 방식이다. 홀수 크기 필터(3×3, 5×5 …)에서 P = (F−1)/2로 깔끔하게 떨어진다.

- 3×3 필터 → P = 1
- 5×5 필터 → P = 2
- 7×7 필터 → P = 3

> VGGNet처럼 Conv에서 크기를 유지(Same)하고 Pooling에서만 크기를 줄이는 설계가 대표적인 활용 예다.

## 5. 출력의 채널(깊이) 수

공식 `O = ⌊(W − F + 2P)/S⌋ + 1`은 **가로·세로(공간) 크기**만 결정한다. 출력의 **채널(깊이)**은 별개이며 다음과 같이 정해진다.

- 필터 하나는 입력의 모든 채널을 보고 **2D feature map 한 장**을 만든다.
- 따라서 **필터를 K개 쓰면 출력 채널이 K개**가 된다.
- 이때 각 필터의 깊이는 입력 채널 수 C와 같아야 한다(필터 형상: F × F × C).

즉 입력 (W × H × C)에 (F × F × C) 필터를 K개 적용하면:

$$
\text{출력 크기} = O_{height} \times O_{width} \times K
$$

---

## 6. 계산 예시

**예시 1 — 기본**

- 입력 5×5, 필터 3×3, 패딩 0, 스트라이드 1
- O = ⌊(5 − 3 + 0)/1⌋ + 1 = 2 + 1 = **3** → 출력 3×3

**예시 2 — Same 패딩**

- 입력 32×32, 필터 3×3, 패딩 1, 스트라이드 1
- O = ⌊(32 − 3 + 2)/1⌋ + 1 = 31 + 1 = **32** → 입력과 동일 (Same)

**예시 3 — 스트라이드 2 (다운샘플링)**

- 입력 7×7, 필터 3×3, 패딩 0, 스트라이드 2
- O = ⌊(7 − 3 + 0)/2⌋ + 1 = ⌊4/2⌋ + 1 = 2 + 1 = **3** → 출력 3×3

**예시 4 — 나눠떨어지지 않는 경우 (내림)**

- 입력 28×28, 필터 5×5, 패딩 0, 스트라이드 3
- O = ⌊(28 − 5)/3⌋ + 1 = ⌊23/3⌋ + 1 = ⌊7.67⌋ + 1 = 7 + 1 = **8** → 출력 8×8

**예시 5 — 채널 포함**

- 입력 32×32×3, 필터 3×3, 패딩 1, 스트라이드 1, 필터 개수 64
- 공간 크기 32×32 유지, 채널은 필터 수 → **출력 32×32×64**

## 7. 파라미터 수 계산

Convolution Layer 한 개의 **학습 파라미터 수**는 출력 공간 크기와 무관하게 다음과 같다(가중치 공유 덕분).

$$
\text{파라미터 수} = (F \times F \times C_{in} + 1) \times K
$$

- F × F × C_in : 필터 하나의 가중치 수 (입력 채널 C_in까지 포함)
- `+ 1` : 필터마다의 편향(bias) 1개
- × K : 필터 개수만큼 반복

**예시:** 입력 채널 3, 필터 3×3, 필터 64개 → (3×3×3 + 1) × 64 = (27 + 1) × 64 = **1,792개**

> 핵심: 파라미터 수는 **입력 이미지의 가로·세로 크기에 의존하지 않는다.** feature map 크기가 커져도 같은 필터를 슬라이딩하며 재사용(parameter sharing)하기 때문이다.

---

## 8. Pooling에도 동일하게 적용

Max/Average Pooling의 출력 크기도 같은 공식을 따른다(보통 패딩 P=0).

$$
O = \left\lfloor \frac{W - F}{S} \right\rfloor + 1
$$

- 예: 입력 32×32, 풀링 2×2, 스트라이드 2 → ⌊(32−2)/2⌋ + 1 = 15 + 1 = **16** → 16×16
- 단, Pooling은 **공간 크기만 줄이고 채널 수는 그대로** 유지하며, 학습 파라미터가 없다.

## 부록. 한눈에 보는 요약

| 항목 | 공식 / 규칙 |
| --- | --- |
| 출력 한 변 크기 | O = ⌊(W − F + 2P) / S⌋ + 1 |
| 출력 채널 수 | 필터 개수 K |
| Same 패딩(S=1) | P = (F − 1) / 2 → 출력 = 입력 |
| Valid 패딩 | P = 0 → 출력 < 입력 |
| Conv 파라미터 수 | (F × F × C_in + 1) × K |
| Pooling 출력 | ⌊(W − F) / S⌋ + 1, 채널·파라미터 변화 없음 |
